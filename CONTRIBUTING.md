# Contribuindo

Obrigado por considerar contribuir com este repositório. Abaixo estão as orientações básicas para facilitar revisão e integração.

1. Abra uma issue antes de começar trabalhos grandes — descreva o problema ou a proposta.
2. Faça um fork e crie uma branch com nome descritivo: `feat/nome-da-funcionalidade` ou `fix/descricao-bug`.
3. Escreva commits claros e pequenos; prefira mensagens no estilo imperativo: `Adiciona validação de entrada`.
4. Inclua testes quando aplicável e garanta que `pytest` passe localmente.
5. Abra um Pull Request descrevendo o que foi alterado e o motivo; vincule a issue correspondente.

Padrões de código e testes
- Mantenha o código legível e com responsabilidade única por módulo.
- Sempre que possível, adicione testes que comprovem o comportamento.

Processo de revisão
- PRs serão revisados por pelo menos uma pessoa. Pode haver pedidos de mudanças.
- Após aprovação, o mantenedor fará o merge.
